AI Agent Specialists

Six specialized agents covering every discipline of offensive security. Each tuned to the right model tier for its domain.

⚡

exploit-dev

Buffer overflows, format strings, web exploits, and deserialization attacks. Automated shellcode generation with target-specific offsets.

/armsforge:exploit buffer-overflow --target windows
🚀

payload-eng

Shellcode generation, loaders, multi-stage delivery systems. Custom encoders with evasion techniques that adapt to target defenses.

/armsforge:payload shellcode --format exe --evasion 2
🤖

implant-dev

C2 framework scaffolding, persistence modules, stealth communication channels. Domain fronting and beacon jitter built in.

/armsforge:loader rust --technique process-hollowing
🔍

opsec-reviewer

Detection risk analysis across AV, EDR, SIEM, and SOC controls. API pattern matching, Sysmon correlation, AMSI trigger analysis.

/armsforge:opsec-review --code ./payload.c
📋

finding-writer

Professional pentest report generation. Executive summaries, technical details, remediation guidance, and risk scoring.

/armsforge:finding --vuln sqli --severity critical
⬆

privesc-analyst

Privilege escalation research for Windows and Linux. Service exploits, token manipulation, kernel vulnerabilities, OSCP/OSEP prep.

/armsforge:privesc --os windows --method service

Platform Features

Comprehensive offensive tooling through Claude Code. MCP tools, templates, snippets, and detection intelligence in one platform.

MCP Integration

Six core tools via Model Context Protocol. Template retrieval, snippet integration, and detection lookup with Zod-validated parameters.

Template Engine

50+ exploit templates covering buffer overflows, process injection, HTTP stagers, and C2 implant skeletons. Ready to customize and deploy.

Code Snippets

25+ reusable patterns for direct syscalls, AES decryption, process injection, and API hooks. Cross-language where possible.

Detection Database

Suspicious Win32 APIs and detection patterns. Sysmon event IDs. ETW providers. AMSI trigger patterns with bypass techniques.

OPSEC Review

Automated detection risk assessment. Static analysis, behavioral analysis, and network evasion guidance before deployment.

Skill Commands

Claude Code slash commands for every workflow. /armsforge:exploit, /armsforge:payload, /armsforge:opsec-review.

Multi-Language

Templates and snippets in C, C++, C#, Python, Rust, Go, and Nim. Language-specific scaffolds optimized for each target.

Agent Routing

Model-tiered agents: Opus for OPSEC analysis, Sonnet for development, Haiku for documentation. Right model for each task.

Exam Support

OSCP and OSEP methodology checklists. Structured privilege escalation approaches. Professional report frameworks.

Platform Capabilities

OPSEC Awareness

Every generated artifact is analyzed for detection risk. Know what evades and what gets caught.

Evasion Guidance

  • String encryption for C2 URLs and credentials
  • Dynamic API resolution to avoid static signatures
  • Alternative API sequences vs. classic injection chains
  • Sleep/jitter to defeat behavioral analysis
  • Domain fronting for network evasion

Detection Coverage

  • Suspicious Win32 API patterns and YARA rules
  • Sysmon event IDs for offensive techniques
  • ETW providers logging injection and hooking
  • AMSI trigger patterns for script-based attacks
  • EDR behavioral signatures and heuristics

Architecture

Four-layer design. MCP bridge, agent orchestration, template engine, and content management. TypeScript throughout.

1

MCP Server Bridge

Node.js MCP server with six registered tools. Zod schema validation on all parameters. Structured JSON output for Claude Code consumption.

2

Agent Orchestration

Six specialized agents with model-tiered routing. Skill-based invocation through Claude Code slash commands. Parallel processing where applicable.

3

Template Engine

Handlebars-based code generation with parameter injection. Template matching by category, language, architecture, and evasion level.

4

Content Management

50+ templates, 25+ snippets, and a detection pattern database. Organized by technique, language, and target platform.

Getting Started

Clone, build, and integrate with Claude Code in minutes.

install.sh 
# Clone and build
$ git clone https://github.com/Real-Fruit-Snacks/armsforge.git
$ cd armsforge

# Install dependencies
$ npm install

# Build for production
$ npm run build

# Run tests
$ npm test
All tests passed
usage — Claude Code 
# List exploit templates
> af_list_templates category="exploit"
Found 12 templates matching criteria

# Generate buffer overflow exploit
> af_get_template name="bof-exploit" target_os="windows"
[template] Generating x64 Windows BOF...
Exploit generated with OPSEC review

# Run OPSEC analysis
> /armsforge:opsec-review --code ./payload.c
Detection risk: LOW
No suspicious API patterns detected
